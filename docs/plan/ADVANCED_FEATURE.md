基于我对tree-sitter高级查询语法的了解和对当前项目实现的分析，我现在可以评估当前服务对高级查询语法的支持程度。

从context7 MCP获取的信息中，我了解到tree-sitter支持以下高级查询语法：

1. **谓词（Predicates）**：
   - `#eq?` - 检查捕获是否等于特定值
   - `#match?` - 检查捕获是否匹配正则表达式
   - `#any-of?` - 检查捕获是否匹配列表中的任何值
   - `#is?` - 检查捕获的特定属性
   - `not-#eq?`、`not-#match?` 等否定谓词
   - `any-#eq?`、`any-#match?` 等量词谓词

2. **锚点（Anchors）**：
   - `.` - 确保子节点是第一个或最后一个命名节点，或兄弟节点是相邻的

3. **交替查询（Alternations）**：
   - `[]` - 定义替代模式，类似于正则表达式中的字符类

4. **指令（Directives）**：
   - `#set!` - 设置键值对
   - `#strip!` - 根据正则表达式从捕获中移除文本
   - `#select-adjacent!` - 过滤捕获文本以保留仅与另一个指定捕获相邻的节点

5. **其他高级功能**：
   - 量词操作符：`+`（一个或多个）、`*`（零个或多个）、`?`（可选）
   - 分组：使用括号分组兄弟节点序列
   - 通配符节点：`(_)` 匹配任何节点

现在让我评估当前项目的实现：

从 [`TreeSitterService.ts`](src/core/TreeSitterService.ts:226-327) 的 `executeQueries` 方法中，我可以看到当前实现：

1. **基本查询支持**：当前实现支持基本的tree-sitter查询，包括节点匹配和捕获
2. **查询创建**：支持通过 `languageModule.query()` 或 `new Query()` 创建查询对象
3. **匹配执行**：使用 `query.matches(tree.rootNode)` 执行查询并获取匹配结果
4. **结果处理**：处理匹配结果并提取捕获的节点信息

但是，当前实现存在以下限制：

1. **谓词支持不完整**：代码中没有明确处理谓词（如 `#eq?`、`#match?` 等），这些谓词可能由底层的tree-sitter库自动处理，但没有在API层面提供额外的支持或验证
2. **指令支持缺失**：没有处理 `#set!`、`#strip!`、`#select-adjacent!` 等指令
3. **高级查询功能未暴露**：没有API层面的支持来验证或处理高级查询语法
4. **查询验证有限**：当前的查询验证只是尝试执行查询，没有专门的语法验证
